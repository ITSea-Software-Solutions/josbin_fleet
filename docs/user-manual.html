Josbin Fleet MS

Complete guide to managing your fleet — vehicles, drivers, services, insurance, fuel, trips, and automated notifications.

📋 Overview

Josbin Fleet MS is a web-based fleet management system designed for Josbin, Paramaribo, Suriname. It gives you a single place to track every vehicle, driver, and maintenance event in your fleet — and automatically alerts you before deadlines are missed.

👤

🔧

🔐 Logging In

Open your browser and go to the app URL (e.g. http://localhost:3000 or your production domain).

http://localhost:3000/login

You will see a login form asking for:

Email Address Required
Your admin email address (default: admin@josbin.sr)
Password Required
Your password (default: password)
⚠️
Change the default password immediately after first login to keep the system secure.

After a successful login you are taken to the Dashboard. Your session stays active until you click Sign Out in the sidebar.

Editing or deleting a vehicle

Click the ✏️ Edit icon on any row to open the edit form. Click 🗑️ Delete to permanently remove the vehicle. Deletion also removes all linked service, insurance, and fuel records.

⚠️
Deleting a vehicle is permanent and cannot be undone. Make sure you have exported any required reports first.

🔔 Notifications

The Notifications page lets you view alert history, trigger scheduled alerts on demand, and send custom one-off notifications.

Summary cards

Four cards at the top show total Sent, Failed, Via Email, and Via WhatsApp counts from the full log history.

Scheduled Notification Check

The Scheduled Notification Check panel shows a live preview of all upcoming deadlines that fall within your configured alert thresholds. It shows which channel (email/WhatsApp) will be used and the configured recipient.

  1. Review the preview tableLists all vehicles/drivers with deadlines due soon, sorted by urgency
  2. Click "Run Notification Check"Sends alerts immediately through the configured channels
  3. See the resultShows how many were sent and how many failed
ℹ️
Notifications also run automatically every day at 08:00 — you do not need to run them manually unless you want to test or send immediately.

❓ Frequently Asked Questions

I'm not receiving email notifications — what do I check?

Go to Settings → Email (SMTP) and verify all fields are filled in correctly. Click Save Settings, then Send Test Email. If it fails, the error message will tell you what's wrong (wrong host, bad credentials, etc.).

WhatsApp messages aren't arriving — why?

If you are using the Twilio sandbox, the recipient must have first opted in by sending join <your-sandbox-code> to +14155238886. Also verify your Twilio Account SID, Auth Token, and sender number in Settings.

Why do I see old alerts on the Dashboard even after renewing a vehicle's insurance?

After renewing, open the vehicle record and update the Insurance Expiry date to the new expiry date. The Dashboard alerts recalculate live based on the dates stored in the system.

Can I assign the same vehicle to multiple drivers?

No — a vehicle can have one assigned driver at a time. To change the driver, edit the vehicle and select a new driver from the dropdown.

How do I reset everything to start fresh?

A system administrator can run the following command on the server:

docker exec josbin_fleet_ms_backend php artisan migrate:fresh --seed
⚠️
This deletes all data and replaces it with demo data. Only use this on a test environment.

How do I add a second admin user?

Currently user management is done through the API or database. A system administrator can create a new user via phpMyAdmin (http://localhost:8081) or by running:

docker exec -it josbin_fleet_ms_backend php artisan tinker

Then in the tinker shell:

App\Models\User::create(['name' => 'New Admin', 'email' => 'new@josbin.sr', 'password' => 'yourpassword']);
